# 第一优先级改动：baostock 扩字段（基本面 + 行业 + 清洗）

按数据源调研报告的「第一优先级」落地：用 baostock 现有接口扩字段，零新增依赖、完全合规。
在你们已做的「市场特征」基础上，新增三类特征。

## 一、改了哪些文件

1. **get_stock_data.py**
   - 日线查询字段追加：`peTTM, pbMRQ, psTTM, pcfNcfTTM, isST, tradestatus`
   - 输出列同步追加这 6 个字段
   - 新增 `get_and_save_industry()`：抓取行业分类，保存为 `data/stock_industry.csv`

2. **code/src/utils.py**（新增 3 个函数，向后兼容）
   - `add_fundamental_features()`：估值特征 —— 原始 PE/PB/PS/PCF + 收益率倒数(ep/bp/sp/cfp) + 当日横截面百分位排名
   - `add_cleaning_features()`：`is_st`、`is_trading` 两个状态特征
   - `add_industry_features()`：行业相对强弱 —— 行业当日均值/标准差、个股相对行业涨跌 `rel_industry_return`、行业内排名、行业上涨占比
   - 共新增 **19** 个特征列（清单见 `FUNDAMENTAL_FEATURE_COLUMNS`）

3. **code/src/train.py & predict.py**
   - 在 `add_market_features` 之后调用上述 3 个函数，并动态把实际存在的新特征列加入 `feature_columns`（与市场特征同样的写法）
   - predict.py 额外加了 **ST/停牌候选过滤**：最新交易日为 ST 或非正常交易的股票不进 Top5（基于原始字段，不受标准化影响；过滤后不足 5 只时自动回退，保证产出）

## 二、向后兼容（重要）

所有新特征都是**优雅降级**的：如果数据里没有这些字段（即还没重新抓数据），相关函数原样返回，`feature_columns` 不变，流程和原来完全一样、现有 `best_model.pth` 也照常能用。
也就是说——**改动不会破坏现状**；要真正用上新特征，需要重抓数据并重训。

## 三、在 Mac 上怎么跑（要重抓数据 + 重训）

新特征会让模型输入维度从 **212 → 231**，所以**旧的 best_model.pth 不再兼容，必须重训**。

```bash
cd Baseline-modified-main
source .venv/bin/activate          # 没有就先 uv sync

# 1. 删除旧数据（避免增量合并时新旧列不一致），重新抓取
rm -f data/stock_data.csv
python get_stock_data.py           # 会同时生成 data/stock_data.csv 和 data/stock_industry.csv

# 2. 重新划分训练/测试集（新列会自动透传）
python data/split_train_test.py

# 3. 重新训练（输入维度变成 231，必须重训）
sh train.sh

# 4. 预测 + 评分
sh test.sh
python test/score_self.py
```

跑完看 `output/result.csv`，参考分数对比 `model/60_158+39/final_score.txt`（改动前是 0.1643）看是否提升。

## 四、几点说明 / 后续可调

- 估值缺失值统一记 0，PE/PCF 为负（亏损）时收益率倒数记 0；如需更精细可改 `add_fundamental_features`。
- ST/停牌过滤目前是「最新日」口径；若想更严格可在训练阶段也排除。
- 行业分类用 baostock `query_stock_industry`（申万口径），存为 `data/stock_industry.csv`，是相对静态的映射表。
- 若验证集 final_score 不升反降，优先怀疑过拟合或估值字段缺失率高，可先只开「基本面」或只开「行业」单独对比。
- 已通过：四个文件 `py_compile` 语法检查；新特征函数用合成数据单测（特征全有限、排名∈[0,1]、行业内相对值均值≈0、旧数据零新增列不报错）。
